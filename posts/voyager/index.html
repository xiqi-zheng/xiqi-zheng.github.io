🧠 VOYAGER: An Open-Ended Embodied Agent with Large Language Models

用 GPT-4 在 Minecraft 里做终身学习的 first-runner：无人类、无梯度、无像素，只靠黑箱提示。

导读

  • 三个模块：自动课程表 + 可执行技能库 + 迭代提示自验证
  • 结果：3.3× 物品、15.3× 科技树、2.3× 探索距离；新地图零样本 100% 成功

核心架构图

三大模块拆解

自动课程表

角色：Prompt 里的导师
输入：当前背包、生态域、昼夜、血饱、附近方块&生物、已/失败任务
输出：一句原子任务，如 Mine 5 coal

技巧两条：

  1. GPT-3.5 先自问自答 5–10 个 Wiki 问题，把答案塞进提示，补充世界知识
  2. warm-up 调度：前 15 个任务只给部分状态，防止提示过长

8 条铁律：别太简单、别太难、可验证、不重复、不建筑、不种植、不交易、单任务

技能库

角色：可插拔函数仓库
技能形式：async function xxx(bot){ ... } 可直接被 Mineflayer 调用

key：功能描述句 → text-embedding-ada-002 向量
value：完整可运行 JS 代码

写入：迭代提示成功 → 自验证通过 → 自动生成描述 → 入库
读取：新任务到来 → GPT-3.5 生成“通用思路句” → 向量 Top-5 → 喂给 GPT-4 参考

复合：函数互相调用，形成技能树，而非扁平宏

迭代提示机制

循环上限：4 轮仍失败 → 课程表直接换新任务
每轮：生成代码 → 执行 → 收集三类反馈 → 反馈写回提示 → GPT-4 重写

三类反馈：

  • 环境反馈：bot.chat() 打印的中间缺失物品、剩余数量
  • 执行错误：JS 运行时异常、API 不存在、参数错误
  • 自验证：另一个 GPT-4 只看最终状态判断 success，失败时给出 critique

自动课程表提出任务 → 迭代提示写代码+自 debug → 成功后将函数存入技能库，下次检索复用，循环不息

实验速览

指标 VOYAGER 最强基线 倍数
独特物品 (160 轮) 63 ~19 3.3×
科技树 (钻石工具) 102 轮 无人解锁 唯一
地图直径 2.3× 更长 - 2.3×
零样本 4 任务 3/3 全部成功 0/3 碾压

消融实验结论

  • 无课程表 → −93% 物品

  • 无技能库 → 后期 plateau

  • 无自验证 → −73% 物品

  • 换 GPT-3.5 → −82% 物品
